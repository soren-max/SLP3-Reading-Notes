# Chapter 1: Introduction

## 章节定位

第三版教材的开篇章节。与传统 NLP 教材按历史发展顺序（规则 → 统计 → 机器学习 → 深度学习）展开不同，本章直接以 Large Language Models 作为起点，确立 LLM 作为现代 NLP 核心范式的定位。

## 核心概念

- **LLM（Large Language Model）**：基于海量文本预训练的大规模语言模型
- **范式转换**：从任务专用模型 → 统一预训练 + 微调 / Prompting
- **基础数据单元**：Token，而非完整句子或语法树

## 例子说明

**LLM 范式 vs. 传统 NLP 范式的对比：**

| 任务 | 传统 NLP 方法 | LLM 方法 |
|---|---|---|
| 情感分析 | 人工设计情感词典 + 特征工程 + SVM/LR | 直接输入评论，Prompt "判断这条评论的情感" |
| 机器翻译 | 独立的统计机器翻译系统（短语表 + 重排序） | 输入 "Translate to Chinese: ..." |
| 命名实体识别 | CRF + 人工设计的词性/大小写特征 | 输入 "提取文本中的人名和地名" |

**例子：** 对句子 "这部电影太棒了" 做情感分析：
- **传统方法**：先分词 → 提取特征（出现"太棒了"正面词 +1）→ 输入 LR 分类器 → 输出正面/负面
- **LLM 方法**：直接 Prompt "请判断以下评论的情感极性：这部电影太棒了" → 模型直接回答 "正面"

两者的本质差异在于：传统方法需要为每个任务重新设计特征和模型，LLM 用同一个模型通过不同的指令完成所有任务。

## 重点总结

- LLM 已成为现代 NLP 的核心范式
- 几乎所有 NLP 任务都可统一在 LLM 框架下完成
- 理解 Token → Embedding → Transformer 的链路是后续学习的基础

## 导师可能提问

**Q1：为什么第三版教材一开始就讲 Large Language Models？**

因为 LLM 已经成为现代 NLP 的核心范式，几乎所有自然语言任务都建立在预训练大模型基础上，因此新版教材直接围绕 LLM 展开。

**Q2：LLM 和传统 NLP 最大区别是什么？**

传统 NLP 更多依赖人工设计规则和统计模型，而 LLM 通过海量数据预训练学习语言规律，能够统一完成多种 NLP 任务。

## 我的理解

LLM 范式的核心贡献不是某一条具体算法，而是**统一了 NLP 的任务范式**——过去做情感分析、命名实体识别、机器翻译需要各自设计不同的模型架构和特征工程，现在同一个预训练模型可以通过不同的 Prompt / Instruction 完成所有这些任务。这种"统一"本身就是一种巨大的进步。

## 与大语言模型 / AI Agent / RAG 的联系

现代 LLM（GPT 系列、Llama、Qwen 等）都是本章所述的 LLM 范式的直接体现。AI Agent 基于 LLM 的推理能力进行工具调用和任务规划；RAG 则利用 LLM 的语言理解能力结合外部知识库进行增强生成。
